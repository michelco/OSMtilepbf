Atlas-OSM


Atlas-OSM est un programme pour visualiser, paramétriser, extraire,… la base OpenStreetMap.
Le principe de base est le découpage en tuiles vectorielles.
Elles sont au format pbf avec un protocole légèrement différent du standard ‘.osm.pbf’
(ajout des coordonnées, des points de clip et des tags entre autres).
 Tags aires :531 Lignes :153 Nœud : 548
Pour avoir les forets complètes à tous les zooms, la base Global Forest Hansen est traitée :
https://earthenginepartners.appspot.com/science-2013-global-forest/download_v1.2.html
Le relief (zoom<14) est SRTM 3s avec les compléments de : 
http://www.viewfinderpanoramas.org/dem3.html
 Le relief (zoom>=14) est AW3D30 (DSM et non DTM correction zoom>14) : 
https://www.eorc.jaxa.jp/ALOS/en/aw3d30/index.htm



Ce programme fonctionne en sphérique et Mercator. Le format des fichiers s’appelle ‘.osmtile.pbf’

J’ai construit des bases réduites vectorielles pour les Zoom 5, 7, 9, 11, 14 puis à partir du zoom 15 c’est la base OSM d’origine.
On peut tout modifier sur le graphisme:
Bouton avec    et Double Click sur un élément y compris changer les icones à partir de SVG.
Les fonts sont indépendantes de Windows et peuvent être modifiées….
J’ai refait les fontes surtout à partir des "Google Noto Fonts" pour traiter toutes les langues en utilisant ICU.
On peut extraire   dans le format pbf d’origine (.osm.pbf) avec sélections de boundaries ou rectangle ou poly.
Puis on peut   réimporter les pbf standards dans l’atlas au format ‘.osmtile.pbf’.
Et plein d’autres choses encore…

Attention : Commencer les extractions par des zones pas trop grandes (admin 8 ou Admin 6) pour tester :
Par exemple les Bouches du Rhône (FR) ID 7393 si les tuiles sont déjà chargées :
Zoom 20: 35s pour extraire en osm.pbf et 68s pour convertir en osmtile.pbf avec CoreI7+SSD
Zoom 14: 7s pour extraire en osm.pbf et 11s pour convertir en osmtile.pbf avec CoreI7+SSD

Installation :
Charger décompresser sur un disque :  OSM_Atlas.zip  (Choix du disque : prévoir un peu de place pour les caches)

Exécuter le programme Atlas_OSM : (OSMtilepbf\PRG\Atlas_OSM.exe)

Sans doute quelques bugs encore, .. la version OSM actuelle est celle du 13/05/24.
Le programme est 32 bits, les programmes d'extractions sont 64bits. Il faut au moins 4 Go de mémoire.
Il faut environ 4 jours de calculs pour traiter une nouvelle base (les réductions sont les plus longues).
Il y aura sans doute une mise à jour environ une fois par mois ou tous les deux mois.
